1. Estrutura geral do componente

O arquivo principal é:

GabaritoTemplate.tsx

Ele gera uma página por aluno contendo:

cabeçalho com informações da prova

QR Code identificando aluno + prova

identificação do aluno

instruções

área OMR com bolhas

marcadores fiduciais (cantos)

Estrutura simplificada:

Página
 ├── Header (informações + QR)
 ├── Área do aluno
 ├── Instruções
 ├── Área OMR
 │    ├── fiducial TL
 │    ├── fiducial TR
 │    ├── fiducial BL
 │    ├── fiducial BR
 │    └── grid de questões
 └── Rodapé

Arquiteturalmente está correto.

Mas existem alguns riscos sérios para leitura automática.

2. Identificação por QR Code

Você colocou:

const qrData = JSON.stringify({
    aid: student.id,
    gid: gabarito.id
});

e gera:

QRCodeSVG size={100}
Pontos positivos

identificação única da folha

evita erro de aluno

permite leitura antes do OMR

Problema importante

QR code dentro do gabarito pode interferir no threshold global da imagem.

Dependendo da câmera ou scanner, o QR gera área muito contrastada.

Recomendação profissional:

Colocar o QR fora da área OMR calibrada.

Exemplo seguro:

┌─────────────────────┐
│ HEADER              │
│         QR          │
├─────────────────────┤
│                     │
│   AREA OMR          │
│                     │
└─────────────────────┘

Nunca misturar QR com região de leitura.

3. Marcadores Fiduciais (âncoras)

Você colocou:

<div class="fiducial tl"></div>
<div class="fiducial tr"></div>
<div class="fiducial bl"></div>
<div class="fiducial br"></div>

Isso é essencial para correção de perspectiva.

Mas existem dois pontos críticos.

Problema 1 — posição relativa

Eles estão dentro do wrapper do gabarito.

Isso significa que qualquer corte ou dobra pode deslocar a referência.

O ideal industrial:

┌───────────────┐
■               ■
│               │
│   GABARITO    │
│               │
■               ■
└───────────────┘

Os marcadores devem ficar nos limites da área OMR.

Problema 2 — formato desconhecido

Seu código não mostra o CSS, mas se os fiduciais forem quadrados simples, a detecção pode falhar.

Padrão profissional:

█ █
█ █

ou

■

com:

tamanho fixo

contraste alto

borda branca ao redor

Tamanho ideal:

10mm a 12mm
4. Organização das questões

Você divide as questões em blocos:

{ start: 0, end: 7 }
{ start: 7, end: 14 }
{ start: 14, end: 21 }
{ start: 21, end: 26 }

Ou seja:

7 + 7 + 7 + 5 = 26 questões.

Isso cria 4 colunas verticais.

Layout visual aproximado:

01 ○ ○ ○ ○ ○
02 ○ ○ ○ ○ ○
...
07 ○ ○ ○ ○ ○

08 ○ ○ ○ ○ ○
...
Isso é bom?

Sim, mas tem um problema.

Se o sistema usa x_centers_pct, colunas precisam ser fixas e previsíveis.

Exemplo profissional:

col1 col2

Ou no máximo:

col1 col2 col3

Quatro blocos aumenta complexidade.

5. Bolhas de marcação

Você gera:

<div class="option-circle"></div>

Sem letra dentro.

Isso é excelente para OMR.

Mas o sucesso depende de três fatores.

1 — tamanho da bolha

Padrão industrial:

5mm a 6mm diâmetro

Menor que isso gera erro.

2 — espessura da borda

Ideal:

0.5mm a 0.8mm

Borda muito fina desaparece na digitalização.

3 — distância entre bolhas

Ideal:

mínimo 2.5x o diâmetro

Se for menor, sombras interferem.

6. Numeração das questões

Você usa:

<span class="q-num">01</span>

Boa prática.

Mas cuidado:

número não pode encostar na bolha.

Distância mínima recomendada:

6mm
7. Grid de questões

Classe usada:

questions-grid-printable-industrial

Se esse CSS estiver usando:

display: grid

isso é bom.

Mas a impressão precisa ser determinística.

Evite:

auto-fit
auto-fill

Use sempre:

grid-template-columns: fixed
8. Impressão

Você usa:

pageBreakAfter: 'always'

Boa prática.

Mas faltam duas coisas essenciais para OMR.

1 — tamanho fixo da página

Deve existir no CSS:

@page {
  size: A4;
  margin: 0;
}
2 — área de segurança

Scanner sempre corta bordas.

Deixe margem:

15mm
9. Problema crítico detectado

O maior risco do template é este:

omr-anchor-wrapper

Se o wrapper não tiver tamanho fixo absoluto, o algoritmo de leitura quebra.

Exemplo correto:

width: 180mm
height: 220mm
position: relative

Sem isso, o warp nunca será consistente.

10. O que está excelente no seu design

Seu template já possui várias boas práticas raras em sistemas educacionais:

✔ QR por folha
✔ identificação automática
✔ marcadores fiduciais
✔ separação de área OMR
✔ bolhas sem texto
✔ grid consistente

Isso já coloca seu projeto muito acima de sistemas escolares comuns.
